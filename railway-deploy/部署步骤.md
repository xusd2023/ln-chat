# LN 聊天室 Railway 部署步骤

## 第一步：准备文件

在 `I:\文件\LN\railway-deploy\` 目录下打开命令行，运行：

```
python prepare.py
```

这会自动生成正确的 `index.html`（修复了硬编码的 localhost 地址）。

---

## 第二步：上传到 GitHub

1. 打开 https://github.com/new 创建新仓库
   - 名字随意，比如 `ln-chat`
   - 选 **Private**（私有，防止别人看到代码）
   - 点 **Create repository**

2. 在 `I:\文件\LN\railway-deploy\` 目录下运行以下命令（把 `你的用户名` 替换为你的 GitHub 用户名）：

```cmd
git init
git add .
git commit -m "init"
git remote add origin https://github.com/你的用户名/ln-chat.git
git push -u origin main
```

---

## 第三步：Railway 部署

1. 打开 https://railway.app 用 GitHub 账号登录

2. 点 **New Project** → **Deploy from GitHub repo**

3. 选择刚才创建的 `ln-chat` 仓库

4. Railway 会自动检测 Node.js 并开始部署（约1~2分钟）

5. 部署完成后点 **Generate Domain**，会得到一个类似：
   ```
   https://ln-chat-production.up.railway.app
   ```
   的固定 HTTPS 域名，把这个链接发给朋友即可！

---

## 文件结构说明

```
railway-deploy/
├── server.js          ← 服务器（已适配云端）
├── index.html         ← 前端页面（运行 prepare.py 后生成）
├── package.json       ← 依赖配置
├── .gitignore
└── prepare.py         ← 一键准备脚本
```

---

## 注意事项

- **数据存内存**：Railway 免费版重启后数据会清空（用户、消息都没了）。这是免费方案的限制。
  - 如果需要数据持久化，可以加 Railway 的 PostgreSQL 插件（仍然免费）
- **文件临时中转**：上传的文件24小时后自动删除，适合临时传文件
- **免费额度**：Railway 每月 $5 额度，小团队完全够用
- **重新部署**：以后改了代码，只需 `git push`，Railway 自动更新
